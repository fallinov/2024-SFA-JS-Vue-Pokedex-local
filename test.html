Timeline Interactive : Chargement d'une Application Vue avec Vuetify, Pinia, et Vue Router

1. Chargement de index.html

Le processus commence avec le chargement du fichier index.html, qui contient la structure de base de l'application et l'élément div#app où l'application Vue sera rendue.

2. Chargement de main.js

Le navigateur charge le fichier main.js, point d'entrée principal de l'application Vue. Ce fichier gère l'initialisation de l'application.

3. Chargement de Vue.js

Vue.js lui-même est chargé. C'est le cœur de l'application, qui permet de gérer les composants et la réactivité.

4. Chargement des éléments (Thèmes, Vuetify, Pinia, Vue Router)

Les thèmes, composants Vuetify, Pinia, et Vue Router sont chargés. Cela inclut la configuration des styles, des données, et du routage de l'application.

5. Montage du composant App.vue

Le composant racine App.vue est monté sur l'élément div#app, prenant le contrôle de l'application Vue.

6. Vue Router détermine le composant à charger

Vue Router vérifie l'URI actuelle et détermine quel composant Vue doit être affiché en fonction de la route.

7. Retour du composant correspondant par Vue Router

Le composant correspondant à la route (par exemple, bulbi.vue) est renvoyé par Vue Router.

8. Chargement du composant retourné par Vue Router

Le composant retourné est chargé et affiché dans la balise de App.vue.

9. Récupération des données dans Pinia

Le composant récupère les données nécessaires à partir du store Pinia, permettant d'accéder à l'état global de l'application.

10. Récupération des composants Vuetify

Les composants Vuetify sont utilisés dans le composant pour créer l'interface utilisateur (ex: v-btn, v-card).

11. Application complète rendue

L'application Vue est entièrement montée et prête à être utilisée, offrant une expérience interactive et dynamique.
